## Decision Framing

The approved G0-A3/Q-5 resolution is strict `L_import > 1.0 p.u.` for four consecutive 15-minute import steps as the primary event, with the same persistent event at `1.1` and `1.2` retained as predeclared sensitivities.

This table does not decide `P_crit`. The current `P_crit = 1e-2` primary and `1e-3` sensitivity are frozen project modelling thresholds unless the PI makes a separate formal amendment.

## Decision Notes

The evidence points toward using `L_import > 1.0 p.u.` for four consecutive 15-minute import steps as the primary event. This is the cleanest definition because it treats nameplate exceedance as congestion while avoiding the unsupported claim that `1.1 p.u.` is the Dutch primary criterion.

The `1.1` and `1.2` thresholds should remain useful sensitivities. They can show how much the conclusion depends on allowing thermal headroom, but they should not be selected after seeing results.

IEC is strong background evidence, but only at the level currently verified: relevance of the transformer loading standard and thermal-loading concepts. Exact IEC table values should be checked against an authorized copy before appearing as manuscript evidence.

Do not let the Betge comparison quietly redefine `P_crit`. Betge is useful for stochastic planning methodology, but its probability space differs from the project's annual episode probability. If the PI wants to change from annual episode probability to overload-hours or episode-rate, that should be a separate formal decision because it changes the core estimand.

Approved Q-5 resolution: amend G0-A3 to primary strict `L_import > 1.0 p.u.` sustained for four consecutive 15-minute import steps over the full planning year, keep `1.1` and `1.2` as predeclared sensitivities, and keep the existing `P_crit` protocol unless separately amended.
