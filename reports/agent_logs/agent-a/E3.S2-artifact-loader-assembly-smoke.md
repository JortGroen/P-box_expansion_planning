# E3.S2 Artifact Loader Assembly Smoke Harness

Task: E3.S2 readiness, stacked draft follow-up on PR #231.

Status: draft/review after PR #231.

## Scope

This stacked unit adds a synthetic-only smoke harness around the accepted-artifact NPZ loader introduced in PR #231. It proves a sequence of guarded component-output artifacts can be loaded into `ComponentAdapterOutput` objects and assembled through the existing IC-1 net-load boundary.

## DID

- Added sequence loading for multiple accepted/synthetic component-output NPZ artifact manifests.
- Added a scaffold-only assembly wrapper that records manifest-ready artifact IDs, array paths, statuses, and explicit no-event/no-probability/no-capacity-screen metadata.
- Added synthetic tests proving baseline, EV, HP, PV, adoption, and flexibility fixture artifacts assemble on one calendar and shared HP/PV weather identity.
- Added a regression test that rejects calendar drift across artifact arrays before assembly.

## Boundaries

This PR is stacked on PR #231 and should remain draft until that upstream branch is merged. No real component arrays were opened. No IC-2 execution, event detection, event counts, threshold evaluation, P(E), capacity/domain screen, G2/G3 conclusion, or manuscript-number analysis was produced.
